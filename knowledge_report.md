## 薄弱知识点（掌握率低于80%）

- **break vs return**：return 结束整个函数，break 只跳出当前循环。在链表的输入循环中误用 return -1 导致打印代码无法执行。
  - 相关表现：输入 -1 后程序直接退出，后面的打印循环没机会运行

- **指针别名（多指针指向同一内存）**：当 `ptail = hp` 后，ptail 和 hp 指向同一块内存，通过 ptail 修改内容，hp 看到的是同一份修改。
  - 相关表现：不理解为什么修改 ptail->poit 会影响 hp 的链表结构

- **链表遍历边界条件**：遍历时先移动指针再访问数据，可能导致 p 为 NULL 时访问 p->ele 产生段错误，或数据多打少打。
  - 相关表现：遍历顺序颠倒导致段错误，-1 标志被多打印出来

## 学习建议
1. **break vs return**：写一个双重循环的 demo 自己跑一遍，用 break 和 return 分别试一下效果
2. **指针别名**：写一段代码，定义两个 int 指针指向同一个变量，通过其中一个修改值，打印另一个观察变化
3. **链表遍历边界**：记住一个口诀 —— "先判断是不是 NULL，再访问；先访问当前节点，再移动指针"
