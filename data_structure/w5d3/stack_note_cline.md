# 栈

## 一、栈的定义

栈（Stack）是一种**操作受限的线性表**，只允许在一端进行插入和删除操作。

- **栈顶（top）**：允许插入和删除的一端
- **栈底（bottom）**：不允许操作的一端

栈的特点是 **先进后出（FILO, First In Last Out）**，也叫 **后进先出（LIFO, Last In First Out）**。

> 生活中的类比：叠盘子——后叠的盘子放在最上面，拿的时候也是先拿最上面的。

---

## 二、栈的基本操作

| 操作               | 功能     | 说明                         |
| ------------------ | -------- | ---------------------------- |
| `CreateEmptyStack` | 创建空栈 | 初始化栈结构                 |
| `PushStack`        | 入栈     | 在栈顶插入元素               |
| `PopStack`         | 出栈     | 删除栈顶元素并返回其值       |
| `TopStack`         | 读栈顶   | 只查看栈顶元素，不删除       |
| `NullStack`        | 判空     | 判断栈是否为空               |
| `FullStack`        | 判满     | 判断栈是否已满（顺序栈需要） |
| `ShowStack`        | 遍历     | 从栈底到栈顶输出所有元素     |
| `LongStack`        | 求长度   | 计算栈中有效元素个数         |
| `DestroyStack`     | 销毁     | 释放栈占用的所有空间         |

## 三、顺序栈

### 3.1 特性

| 维度     | 描述                           |
| -------- | ------------------------------ |
| 逻辑结构 | 线性结构（一对一）             |
| 存储结构 | 顺序存储（数组）               |
| 操作特点 | 只在一端操作，栈顶下标控制全部 |

### 3.2 结构体定义

```c
typedef struct sequerntialstack
{
    datatype *data;   // 动态数组首地址
    int maxlen;       // 数组容量
    int top;          // 栈顶下标（最后一个有效元素的下标）
}ss, *sp;
```

为什么用 `datatype *data` 而不是 `datatype data[N]`？
- 动态数组：空间大小由 `CreateEmptySeqStack(len)` 参数决定，灵活
- 静态数组：空间写死在结构体里，不够灵活

### 3.3 top 指针的含义

```
初始状态：top = -1  （栈空）
入栈：    top++  ->  data[top] = x
出栈：    x = data[top]  ->  top--
读栈顶：  x = data[top]  （top 不变）
判空：    top == -1
判满：    top == maxlen - 1
长度：    top + 1
```

示意图（maxlen=8）：

```
初始：  top = -1
        [  ] [  ] [  ] [  ] [  ] [  ] [  ] [  ]

入栈1：  top = 0
        [ 1 ] [  ] [  ] [  ] [  ] [  ] [  ] [  ]
入栈2：  top = 1
        [ 1 ] [ 2 ] [  ] [  ] [  ] [  ] [  ] [  ]
入栈3~7：
        [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ 6 ] [ 7 ] [  ]
                                              top=6

出栈：  返回 data[6] = 7, top = 5
        [ 1 ] [ 2 ] [ 3 ] [ 4 ] [ 5 ] [ 6 ] [ 7 ] [  ]
                                    top=5
（注意：出栈后 data[6] 还存着 7，但 top=5 意味着它不再是有效数据了）
```

### 3.4 顺序栈的遍历问题

**你在顺序栈中写了两个遍历函数，这是对的：**

```c
// 遍历整个数组（包括未存放数据的空间）
void ShowSepStack(sp p)
{
    for (int i = 0; i < p->maxlen; i++) { ... }
}

// 只遍历有效数据（从0到top）
void RealShowSepStack(sp p)
{
    for (int i = 0; i <= p->top; i++) { ... }
}
```

- `ShowSepStack` 用于调试，可以看到数组全部空间的情况
- `RealShowSepStack` 才能反映栈的真实内容

但在你 `data_structure/w5d3/2/1/sequentialstack.c` 里的 `ShowStack`（之前的那个代码）就只写了一个"遍历整个数组"的版本，这是Bug——应该区分或只用 `<= top`。

### 3.5 顺序栈的优点和缺点

| 优点                        | 缺点                               |
| --------------------------- | ---------------------------------- |
| 实现简单，代码量少          | 需要预先知道最大容量               |
| 内存连续，访问速度快        | 扩容需要 realloc，可能搬移所有数据 |
| 随机存取，O(1) 访问任意位置 | 固定容量，空间不足时需要手动扩容   |

## 四、链式栈

### 4.1 特性

链式栈本质是 **单向链表 + 只允许头插头删**。

| 维度     | 描述                                 |
| -------- | ------------------------------------ |
| 逻辑结构 | 线性结构（一对一）                   |
| 存储结构 | 链式存储（单链表）                   |
| 入栈操作 | 头插法（在头节点后面插入）           |
| 出栈操作 | 头删法（删除头节点后面的第一个节点） |

### 4.2 结构体定义

```c
typedef struct linkedstack
{
    datatype data;
    struct linkedstack *next;  // 注意：next 是习惯命名！
}sl, *sp;
```

### 4.3 入栈过程（头插法）

```
初始状态：  *top = NULL

入栈 A：    p->data = A, p->next = *top (= NULL), *top = p
            [ A | NULL ]
              ^
              top

入栈 B：    p->data = B, p->next = *top (= A节点), *top = p
            [ B | * ] -> [ A | NULL ]
              ^
              top

入栈 C：    p->data = C, p->next = *top (= B节点), *top = p
            [ C | * ] -> [ B | * ] -> [ A | NULL ]
              ^
              top
```

### 4.4 出栈过程（头删法）

```
出栈前：    top -> [ C | * ] -> [ B | * ] -> [ A | NULL ]

出栈：      l = *top          // 保存 C 节点地址
            data = l->data    // 取数据 C
            *top = (*top)->next  // top 指向 B
            free(l)           // 释放 C 节点

出栈后：    top -> [ B | * ] -> [ A | NULL ]
```

### 4.5 判空条件

```c
int NullStack(sp top)
{
    return top == NULL;
}
```

链式栈判空就是栈顶指针是否为 NULL。注意这里不需要像顺序栈那样判满——链式栈是动态分配空间的，理论上只受堆内存总量限制。

### 4.6 链式栈的优点和缺点

| 优点                       | 缺点                   |
| -------------------------- | ---------------------- |
| 无需预先确定容量，动态扩展 | 每个节点多一个指针开销 |
| 入栈出栈都是 O(1)          | 内存不连续，有碎片     |
| 不存在"满"的问题           | 实现比顺序栈复杂一点   |

---

## 五、顺序栈 vs 链式栈

| 对比维度 | 顺序栈                | 链式栈             |
| -------- | --------------------- | ------------------ |
| 底层结构 | 数组                  | 单链表             |
| 存储空间 | 连续内存              | 非连续内存         |
| 容量问题 | 固定容量 / 需手动扩容 | 动态分配，无限扩展 |
| 入栈     | top++ 后赋值 O(1)     | 头插法 O(1)        |
| 出栈     | top-- O(1)            | 头删法 O(1)        |
| 判满     | top == maxlen-1       | 不需要             |
| 判空     | top == -1             | top == NULL        |
| 遍历     | 连续内存，缓存友好    | 逐个指针跳转，稍慢 |

## 六、栈的典型应用

1. **函数调用栈**：C程序的函数调用和返回就是靠栈实现的
   - 调用函数时，将返回地址、局部变量压栈
   - 函数返回时，从栈顶弹出恢复上下文

2. **表达式求值**：编译器将中缀表达式（`a+b*c`）转后缀表达式需要栈

3. **浏览器的后退**：每访问一个新页面就压栈，后退就是出栈

4. **括号匹配**：`{ [ ( ) ] }` 的匹配检查，遇到左括号入栈，遇到右括号出栈检查

5. **撤销操作**：编辑器 Ctrl+Z 就是用栈实现的

---

## 七、总结

```
栈的本质：操作受限的线性表，只在一端操作

顺序栈：数组 + top 指针
  - 入栈：top++ ; data[top] = x
  - 出栈：x = data[top] ; top--
  - 判空：top == -1
  - 判满：top == maxlen - 1

链式栈：单链表 + 只允许头插头删
  - 入栈：头插法
  - 出栈：头删法
  - 判空：top == NULL

共同原则：先进后出（FILO / LIFO）
```

你之前写的顺序栈代码整体是标准的，唯一要注意的是 `ShowStack` 遍历时应该区分"遍历整个数组"和"只遍历有效数据"——你后来在 `w5d3/` 下（带有 `RealShowSepStack` 的版本）已经意识到了这个问题，做了两个版本，这个思路是对的。链式栈的 `poit` 建议以后统一改成 `next`，因为 `next` 是链表领域的通用命名规范。
