# 数据结构整体框架

## 一、为什么学数据结构

你的 `superstruct.md` 里写得很对：

> **数据结构 + 算法 = 程序**

这句话是图灵奖得主 Niklaus Wirth 的名言。学数据结构的目的是：

1. **写更高效的程序**——同样的功能，用合适的数据结构可以快成百上千倍
2. **解决问题有章可循**——遇到实际问题时，能判断该用什么方式来组织数据
3. **为后续学习打基础**——Linux 内核、嵌入式系统、AI 框架，底层全是用这些数据结构

## 三、逻辑结构（重点）

逻辑结构描述的是**数据元素之间的逻辑关系**，不关心数据在内存中的物理位置。

### 3.1 线性结构（你已学完）

线性结构的特点是：数据元素之间是一对一的关系，除了第一个和最后一个，每个元素都有唯一的直接前驱和直接后继。

```
顺序表：  [A] -- [B] -- [C] -- [D] -- [E]
链表：    [A] -> [B] -> [C] -> [D] -> [E]
栈：      只能从一端操作，FILO
          入栈顺序：A -> B -> C -> D
          出栈顺序：D -> C -> B -> A
队列：      从一端入、另一端出，FIFO
          入队顺序：A -> B -> C -> D
          出队顺序：A -> B -> C -> D
```

**四种线性表的对比：**

| 数据结构 | 操作端 | 进出规则 | 操作受限？ |
|---------|--------|---------|-----------|
| 顺序表 | 任意位置 | 无限制 | 无 |
| 链表 | 任意位置 | 无限制 | 无 |
| 栈 | 同一端 | FILO/LIFO | 是（只能在一端操作） |
| 队列 | 两端 | FIFO | 是（队尾入、队头出） |

**为什么栈和队列叫做"操作受限的线性表"？**
因为栈和队列本质上是链表/顺序表实现的功能子集——栈只允许头插头删，队列只允许尾接头删。限制操作范围反而让它们更适合特定的场景（如函数调用、消息队列）。

### 3.2 树形结构（预告）

树形结构的特点是：数据元素之间是一对多的关系，有一个根节点，根节点下有若干子节点。

```
         [A]           <- 根节点
        /   \
      [B]   [C]        <- 子节点
      / \     \
    [D] [E]   [F]      <- 叶子节点（没有子节点）
```

你会学到：
- 二叉树的性质和遍历（前序、中序、后序、层序）
- 二叉搜索树（BST）
- 堆（完全二叉树，用于实现优先级队列）

### 3.3 图状结构（预告）

图的特点是：数据元素之间是多对多的关系，任意两个节点之间都可能存在连接。

```
    [A] --- [B]
     | \     |
     |  \    |
    [C] --- [D]
```

你会学到：
- 图的存储（邻接矩阵、邻接表）
- 图的遍历（DFS 深度优先 / BFS 广度优先）
- 最小生成树、最短路径

## 四、存储结构（你已接触两种）

存储结构是逻辑结构在计算机内存中的具体实现方式。

### 4.1 顺序存储

用一组**连续的存储单元**存储数据元素。

```c
// 顺序表
int data[64];   // 内存中连续存放：data[0], data[1], ... data[63] 地址连续

// 顺序栈
datatype *data = malloc(sizeof(datatype) * 8);  // 堆上连续空间

// 循环队列
datatype *data = malloc(sizeof(datatype) * 8);  // 堆上连续空间
```

**优点：** 随机存取 O(1)，内存连续缓存友好
**缺点：** 容量固定，插入删除需要移动元素 O(n)

### 4.2 链式存储

用一组**任意的存储单元**存储数据元素，通过指针建立逻辑关系。

```c
// 链表节点
struct node {
    datatype data;
    struct node *next;  // 指针指向下一个节点
};

// 链式栈
struct linkedstack {
    datatype data;
    struct linkedstack *next;
};
```

**优点：** 动态扩展，插入删除 O(1)
**缺点：** 不能随机存取，每个节点多一个指针开销

### 4.3 选择原则

| 场景 | 推荐存储方式 |
|------|-------------|
| 频繁查找、较少插入删除 | 顺序存储 |
| 频繁插入删除、较少查找 | 链式存储 |
| 数据量固定、内存紧张 | 顺序存储 |
| 数据量不确定、需要动态扩展 | 链式存储 |
| 对实时性要求高（不能有realloc延迟） | 链式存储 |

## 五、算法复杂度基础

### 5.1 时间复杂度的概念

```
T(n) = O(f(n))
```

- `T(n)`：实际执行时间（是 n 的函数）
- `O`：大O符号，表示时间规模的数量级
- `n`：问题规模（数据量大小）
- `f(n)`：算法中语句重复执行的次数

### 5.2 常见的时间复杂度

| 复杂度 | 名称 | 示例 | 数据量 1000 时的操作数 |
|--------|------|------|----------------------|
| O(1) | 常数阶 | 数组访问 arr[5] | 1 |
| O(log n) | 对数阶 | 二分查找 | ~10 |
| O(n) | 线性阶 | 遍历查找 | 1000 |
| O(n log n) | 线性对数阶 | 快速排序 | ~10000 |
| O(n^2) | 平方阶 | 冒泡排序 | 1,000,000 |
| O(2^n) | 指数阶 | 斐波那契递归 | 天文数字 |

### 5.3 你已学过的代码的时间复杂度

| 操作 | 代码 | 时间复杂度 |
|------|------|-----------|
| 顺序表插入 | `seplist_insert` | O(n) |
| 顺序表删除 | `seplist_delete` | O(n) |
| 链表插入 | `Insert` | O(n)（因为要找到位置） |
| 链表头插 | `InsertHead` | O(1) |
| 链表删除 | `delete` | O(n) |
| 链表逆转 | `Reverse` | O(n) |
| 顺序栈入栈 | `PushStack` | O(1) |
| 顺序栈出栈 | `PopStack` | O(1) |
| 链式栈入栈 | `PushStack` | O(1) |
| 链式栈出栈 | `PopStack` | O(1) |
| 循环队列入队 | `EnQueue` | O(1) |
| 循环队列出队 | `DeQueue` | O(1) |

注意：**链表按值查找、指定位置插入/删除的时间复杂度是 O(n)**，因为需要遍历找到目标位置。而**栈和队列的所有操作都是 O(1)**，这是它们"操作受限"的回报——限制了你能操作的地方，换来了恒定的效率。

## 七、总结

```
数据结构 = 逻辑结构 + 存储结构 + 操作

逻辑结构决定了"数据之间什么关系"
    |
    +-- 线性（一对一）：顺序表、链表、栈、队列
    +-- 树形（一对多）：二叉树、堆

    |
    +-- 顺序存储：数组（连续）

操作决定了"你能对它干什么"
    |
    +-- 增删改查（所有结构都有）
    +-- 操作受限的限定（栈和队列在一端或两端操作）

  - 顺序表（数组+last）
  - 单向链表（头插尾插、指定位置插入删除、按值/按位置查找、逆转）

  - 顺序栈和链式栈（FILO）
  - 循环队列和链式队列（FIFO）
