# 队列

## 一、队列的定义

队列（Queue）是一种**操作受限的线性表**，它只允许在表的一端进行插入、在另一端进行删除。

- **队头（front）**：允许删除的一端
- **队尾（rear）**：允许插入的一端

队列的特点是 **先进先出（FIFO, First In First Out）**。

> 生活中的类比：食堂打饭排队——新来的人站到队伍末尾（入队），打完饭的人从队伍最前面离开（出队）。先来的人先打到饭。

---

## 二、队列的基本操作

| 操作          | 功能                 | 对应栈的操作            |
| ------------- | -------------------- | ----------------------- |
| `CreateQueue` | 创建空队列           | `CreateStack`           |
| `EnQueue`     | 入队（队尾插入）     | `PushStack`（栈顶插入） |
| `DeQueue`     | 出队（队头删除）     | `PopStack`（栈顶删除）  |
| `EmptyQueue`  | 判空                 | `NullStack`             |
| `GetFront`    | 取队头元素（不出队） | `GetTop`（取栈顶元素）  |
| `ShowQueue`   | 遍历                 | `ShowStack`             |
| `LengthQueue` | 求长度               | `LongStack`             |

---

## 三、队列和栈的本质对比

| 对比维度       | 栈                   | 队列                       |
| -------------- | -------------------- | -------------------------- |
| 数据进出规则   | 后进先出（LIFO）     | 先进先出（FIFO）           |
| 插入和删除位置 | 同一端（栈顶）       | 两端（队尾插入、队头删除） |
| 核心操作       | push / pop           | enqueue / dequeue          |
| 想象场景       | 叠盘子、浏览器的后退 | 排队打饭、打印机任务队列   |

## 四、顺序队列的问题——假溢出

### 4.1 普通顺序队列

用数组实现队列时，结构体通常包含：

```c
#define MAX_SIZE 8
typedef struct queue
{
    datatype data[MAX_SIZE];
    int front;  // 队头下标
    int rear;   // 队尾下标
}queue_t;
```

- 初始状态：`front = 0, rear = 0`
- 入队：`data[rear] = x; rear++`
- 出队：`x = data[front]; front++`

### 4.2 假溢出问题

假设数组容量为8，执行以下操作：

```
1. 入队 5 次  -> front=0, rear=5
2. 出队 5 次  -> front=5, rear=5  （队列空了，存了0个元素）
3. 入队 3 次  -> front=5, rear=8  （rear 到数组末尾了）
```

此时 `rear == MAX_SIZE`，所以认为队列已满，不能再入队了。

但你看数组的前面（`data[0]~data[4]`）其实是空的，白白浪费了空间。这就是 **假溢出**——数组还有空间，但 rear 指针已经走到头了。

> 这不是"真正的满了"，而是"指针走到头了"。

## 五、循环队列——解决假溢出

### 5.1 核心思想

把数组想象成一个**环**，数组的最后一个元素后面就是第一个元素。rear 和 front 走到尽头后，通过 **取模运算** 跳回开头。

```c
// 移动 rear/front 时：
rear  = (rear + 1) % MAX_SIZE;
front = (front + 1) % MAX_SIZE;
```

### 5.2 如何知道"满"和"空"

因为把数组做成环了，你需要区分队列是空还是满：

**状态一的区分方案：**
- 空队列：`front == rear`
- 满队列：`(rear + 1) % MAX_SIZE == front`

用 **牺牲一个元素空间** 来区分——队列满时，rear 紧挨在 front 后面，中间空一个位置不存数据。

> 换句话说：循环队列最多存 `MAX_SIZE - 1` 个元素，最后那个位置是"哨兵"，专门用来区分空和满。

### 5.3 循环队列的完整工作过程

假设 MAX_SIZE = 5

```
初始状态：  front=0, rear=0
            0    1    2    3    4
          [  ] [  ] [  ] [  ] [  ]
          f/r

入队 A:    rear <-- data[0]=A, rear=(0+1)%5=1
          [ A ] [  ] [  ] [  ] [  ]
          f     r

入队 B:    rear <-- data[1]=B, rear=(1+1)%5=2
          [ A ] [ B ] [  ] [  ] [  ]
          f           r

入队 C:    rear <-- data[2]=C, rear=(2+1)%5=3
          [ A ] [ B ] [ C ] [  ] [  ]
          f                 r

入队 D:    rear <-- data[3]=D, rear=(3+1)%5=4
          [ A ] [ B ] [ C ] [ D ] [  ]
          f                       r

此时 (rear+1)%5 = (4+1)%5 = 0 == front，所以队列已满
（实际存了4个元素，容量5，牺牲1个位置）
          [ A ] [ B ] [ C ] [ D ] [  ]
          f                       r

出队 A:    data = data[front] = A, front=(0+1)%5=1
          [ A ] [ B ] [ C ] [ D ] [  ]
                f                 r

出队 B:    data = data[front] = B, front=(1+1)%5=2
          [ A ] [ B ] [ C ] [ D ] [  ]
                      f           r

入队 E:    rear <-- data[4]=E, rear=(4+1)%5=0
          [ A ] [ B ] [ C ] [ D ] [ E ]
                      f     r

此时 rear 回到开头，继续复用前面的空间！
```

这就是循环队列的 **核心魅力**：数组空间被循环使用，不再浪费。

### 5.4 循环队列的判空和判满公式

| 条件     | 公式                         |
| -------- | ---------------------------- |
| 空队列   | `front == rear`              |
| 满队列   | `(rear + 1) % max == front`  |
| 队列长度 | `(rear - front + max) % max` |

注意长度公式里的 `+ max` 是为了处理负数情况。

## 六、链式队列

链式队列本质上就是 **带头节点的单链表 + 尾插法入队 + 头删法出队**。

### 6.1 结构体定义

```c
typedef struct node
{
    datatype data;
    struct node *next;
}node_t, *node_p;

typedef struct queue
{
    node_p front;  // 指向头节点（队头）
    node_p rear;   // 指向尾节点（队尾）
}queue_t, *queue_p;
```

为什么用两个指针？（front 和 rear）
- 入队在队尾，用 `rear` 可以直接在尾部插入 O(1)
- 出队在队头，用 `front` 可以快速定位头节点 O(1)
- 如果只用一个指针，另一端的操作需要遍历链表，变成 O(n)

### 6.2 各操作本质

| 操作          | 本质                                           |
| ------------- | ---------------------------------------------- |
| `CreateQueue` | 创建头节点，front=rear=指向头节点              |
| `EnQueue`     | 尾插法：在 rear 后面插入新节点，然后 rear 后移 |
| `DeQueue`     | 头删法：删除 front->next 节点                  |
| `EmptyQueue`  | front == rear 或 front->next == NULL           |
| `ShowQueue`   | 从 front->next 开始遍历到 NULL                 |

### 6.3 链式队列 vs 循环队列

| 对比维度   | 循环队列           | 链式队列               |
| ---------- | ------------------ | ---------------------- |
| 空间利用率 | 牺牲1个元素空间    | 每个节点多一个指针开销 |
| 动态扩容   | 需要手动 realloc   | 天然动态，无需扩容     |
| 内存碎片   | 连续内存，无碎片   | 节点分散，有碎片       |
| 出队       | 下标移动 O(1)      | 释放节点 O(1)          |
| 适用场景   | 数据量固定、嵌入式 | 数据量可变、通用场景   |

## 七、队列的典型应用场景

1. **打印机任务队列**：多个文档排队打印，先提交的先打印
2. **消息队列（MQ）**：进程间通信的典型方式，生产者放入消息、消费者取出消息
3. **BFS广度优先搜索**：树的层序遍历、图的广度遍历都依赖队列
4. **IO缓冲区**：键盘输入缓冲区、网络数据接收缓冲区
5. **操作系统进程调度**：先来先服务（FCFS）调度算法

---

## 八、总结时间线

```
栈（LIFO）    -->   单端操作（只有栈顶）
队列（FIFO）  -->   双端操作（队头出、队尾入）
                 -->   数组实现 -> 假溢出问题
                 -->   循环队列（取模解决）
                 -->   链表实现（天然无溢出）
```

你接下来应该先写 **循环队列** 的代码实现，因为取模运算和判满条件是队列最重要的考点。写通了循环队列，链式队列就只是链表操作加个 front/rear 封装，会轻松很多。
